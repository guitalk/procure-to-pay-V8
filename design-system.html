Design System Reference
Source-of-truth tokens, type, and components for the demo + production scaffold.
How to use this — Every token below maps 1:1 to a CSS variable in styles.css. When you scaffold the Tailwind config in the Next.js app, copy these values exactly. The dashboard at index.html is the live reference for how they compose. Shadcn primitives are the target component layer.

Frontend Next.js (App Router) + TypeScript + Tailwind + Shadcn UI
Typeface IBM Plex Sans (UI) · IBM Plex Mono (IDs, numbers, tabular)

Density Comfortable. Optimized for data-heavy tables but not cramped.

6 · Handoff notes
Shell first. The left nav + top bar + context switcher in index.html is the layout for every authenticated page. Build it once as a Next.js layout component.
ID display format. Always render IDs as PR-2026-105 / RFQ-2026-042 / PO-2026-018 in IBM Plex Mono, accent color, medium weight. The TSD stores them as integers — format on the application layer.
Indian numbering. Use lakhs and crores in display (₹15,47,500 not ₹1,547,500). Use IBM Plex Mono with font-feature-settings: "tnum" 1 for any numeric column to keep digits aligned.
No "AI" labels in the UI. AI moments (cost normalization, deviation flagging, line-item extraction) appear as smart defaults — never as branded callouts. The product positions as sophisticated, not "AI-powered."
Soft delete everywhere. Per the FSD, every reject/delete action opens a modal that requires an internal comment before submit. Build a reusable <ConfirmDestructive comment-required> primitive.
Project-context filter is global. The "Active Context · Client · Project" selector in the top bar gates everything below it. All API calls must include the active project_id.
Subtle borders > heavy borders. Use --border-subtle for row dividers inside cards, --border for card outlines, --border-strong only on focused inputs or active states.
